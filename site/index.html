CrowdTensor

Turn ordinary machines into a fault-tolerant AI workload swarm. The current Alpha validates the daemon, protocol, and recovery mechanics before real GPU aggregation and LLM inference arrive.

What Works Today

CrowdTensorD is a working Alpha control plane for controlled local and remote demos. It is intentionally small, deterministic, and CPU-friendly so reliability can be tested anywhere.

Alpha ready

Coordinator / Miner Loop

Task claim, heartbeat, result upload, timeout requeue, stale result rejection, and checkpoint replay.

Alpha ready

Remote Miner Demo

Token-backed admission, invite generation, `/ready` preflight, retries, and operator diagnostics.

Experimental

Browser Experiments

WebRTC tensor tunnel, browser Worker compute probe, and browser Miner bridge smoke tests.

What Is Not Ready

Not yet

Real LLM Inference

The model bundle path is a contract rehearsal, not production Swarm Inference.

Not yet

P2P Public Network

Not yet

GPU / WebGPU Runtime

CPU-only acceptance remains the baseline while GPU and browser compute adapters are designed.

Run A Local Demo

Install, run the First-run Doctor, start the Coordinator, and launch one local Miner.

python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
python3 scripts/doctor.py --json
crowdtensord --host 127.0.0.1 --port 8787 --state-dir state

In another shell:

crowdtensor-miner \
  --coordinator http://127.0.0.1:8787 \
  --miner-id local-1 \
  --once
